# План: удалённый доступ (будущая фаза)

Этот документ — замороженный план, который закрывает вопрос «как дотянуться с телефона с 4G/другого wi-fi до своего ПК, когда дома тебя нет».
Текущий релиз работает **только в локальной сети**. Удалённый доступ отложен — включится отдельной фазой, когда до него дойдут руки.

## Постановка задачи

Пользователь сидит вне дома, ПК дома включён с запущенным `remotelauncher`. Нужно, чтобы телефон нашёл этот ПК и запустил программу одним тапом. Без того чтобы:

- ставить Tailscale / Cloudflare / что-то отдельное
- лезть руками в настройки роутера
- держать статический IP

## Что не подошло (и почему)

**UPnP + DDNS (изначально задуманный B4).**
- UPnP умеет попросить роутер открыть порт — но у многих провайдеров (особенно мобильных) стоит **CGNAT**, где UPnP физически не может дать внешний адрес: сервер провайдера прячет клиента за общим IP. Для них вариант мёртв.
- DDNS даёт постоянное имя для меняющегося IP провайдера — но требует аккаунта у DuckDNS/No-IP/etc и одноразовой настройки. То есть «plug-and-play» не получается.
- У многих домашних провайдеров IP меняется нечасто (раз в месяц), но всё равно меняется, и клиент без DDNS начинает давать «не подключается».

Итог: UPnP+DDNS решает задачу только для части пользователей и не является «настоящим zero-config».

## Принятая модель: туннель через VPS автора

Сервер на ПК держит **постоянное исходящее соединение** к VPS, который держит автор. Телефон ходит не на домашний IP, а на VPS — VPS пересылает трафик обратно в туннель на ПК.

### Что это даёт

- **Работает у всех**, включая CGNAT и мобильный интернет — исходящее наружу почти всегда разрешено.
- **Безопаснее**: домашний роутер снаружи чист, никаких открытых портов, ботнетам нечего сканировать.
- **Стабильный адрес**: клиент знает короткое имя вида `<id>.remotelauncher.app`. Реальный IP провайдера может хоть каждый день меняться — пользователю всё равно.
- **UPnP/DDNS не нужны вообще.**

### Цена

- Latency чуть выше — трафик крутится через VPS. Для запуска программ по тапу (запрос на сотню байт) неощутимо.
- Нагрузка и трафик на VPS растут с числом пользователей. Поэтому модель — **платная подписка**: автор держит VPS на деньги подписчиков.
- Если VPS лёг — удалённый доступ у всех подписчиков отвалился. Локальная сеть продолжает работать как раньше.
- Если автор уйдёт — платный функционал умрёт. Поэтому надо продумать путь миграции (например, документированный способ поднять свой туннель-сервер).

## Два тира

| Тир | Что работает | Цена |
| --- | --- | --- |
| **Free** | Только локальная сеть. Весь текущий функционал: пара, ярлыки, приложения, веб-админка. | 0 |
| **Paid** | Всё из Free + удалённый доступ через туннель. Один раз ввёл ключ в веб-админку — и работает везде. | Подписка |

Бесплатный тир остаётся функционально полным для домашнего сценария. Платный — только для тех, кто хочет дотягиваться из любой точки.
