# RemoteLauncher — Техническое задание

## Суть проекта

Приложение для запуска программ на Linux-десктопе с Android-смартфона из любой точки (интернет, локальная сеть). Не удалённый рабочий стол, а лаунчер — видишь список приложений, нажал — запустилось на ПК.

## Компоненты

### 1. Сервер (Linux, Go)

- Один бинарник, без зависимостей
- Парсит `.desktop` файлы из стандартных путей (`/usr/share/applications/`, `~/.local/share/applications/`)
- Отдаёт список приложений с иконками через REST API
- Запускает/останавливает приложения
- Отслеживает статус (запущено/нет) через WebSocket
- Опциональная регистрация в локальной сети через mDNS (Avahi)

### 2. Клиент (Android, Kotlin)

- Подключение к серверу по адресу (IP/домен + порт)
- Автообнаружение в локальной сети через mDNS (бонус)
- Показывает сетку приложений с иконками (Material Design)
- Запуск одним нажатием, статус в реальном времени
- Возможность остановить запущенное приложение
- Распространение: APK через GitHub Releases

## API сервера (REST + WebSocket)

| Метод | Путь | Описание |
|-------|------|----------|
| GET | /api/apps | Список приложений |
| GET | /api/apps/{id}/icon | Иконка приложения |
| POST | /api/apps/{id}/launch | Запустить |
| POST | /api/apps/{id}/stop | Остановить |
| GET | /api/status | Статус сервера |
| WS | /ws | Статус приложений в реальном времени |

## Подключение

Основной режим — подключение по адресу (домен/IP) через интернет.

Способы доступа снаружи (на выбор пользователя):
1. Проброс порта на роутере + DDNS
2. Reverse proxy (nginx/Caddy)
3. Tailscale/ZeroTier — без настройки роутера
4. Cloudflare Tunnel

mDNS остаётся как бонус для автообнаружения в локальной сети.

## Безопасность

- TLS обязателен (Let's Encrypt или self-signed)
- Первое подключение — pairing через PIN-код
- Далее — сохранённый токен
- Rate limiting на авторизацию
- Опциональный whitelist IP
- Сервер по умолчанию слушает все интерфейсы (настраивается)

## MVP (v1)

1. Сервер отдаёт список приложений с иконками
2. Клиент подключается по адресу, показывает список
3. Запуск приложения по нажатию
4. PIN-pairing при первом подключении
5. HTTPS

## Следующие версии (v2+)

- Статус запущенных приложений в реальном времени (WebSocket)
- Остановка приложений
- Категории и поиск
- Избранные приложения
- Виджет на home screen Android
- mDNS-автообнаружение в локальной сети
